# [04주차] 미래컴퓨터와 양자컴퓨팅 기술

## 1. 오늘의 키워드
- 미래의 컴퓨터
- 양자컴퓨터(Quantum Computer), 큐비트(Qubit)
- 중첩(Superposition), 얽힘(Entanglement)
- 양자컴퓨팅의 활용과 한계

## 2. 강의 핵심 내용 요약
- 미래의 컴퓨터는 단순히 더 빠른 CPU가 아니라, **GPU, NPU(신경망 프로세서), 양자컴퓨터** 등 다양한 형태의 연산 장치가 공존하는 이기종 컴퓨팅 환경으로 발전할 것이라 설명했다.  
- 양자컴퓨터는 0 또는 1만 가지는 기존 비트(bit) 대신, 0과 1이 동시에 존재할 수 있는 **큐비트(qubit)** 를 사용하며, 중첩과 얽힘이라는 양자역학적 특성을 활용해 기존 컴퓨터와는 다른 방식의 연산을 수행한다.    
- 다만 현재 양자컴퓨터는 큐비트 수가 충분히 크지 않고, 노이즈와 에러율 문제, 냉각·하드웨어 비용 등의 한계 때문에 **“당장 범용 상용화”보다는 실험·연구 단계**에 더 가깝다는 점도 함께 설명했다.  
- 강의에서는 양자컴퓨팅이 기존 컴퓨터를 완전히 대체한다기보다는, **특정 문제(암호 해독, 최적화, 시뮬레이션 등)에 특화된 보조적인 역할**을 할 가능성이 크다고 정리했다.

## 3. 인상 깊었던 포인트
- 기존 컴퓨터가 더 빨라질수록 언젠가는 한계에 부딪힌다는 이야기를 많이 들었는데, 그 한계를 넘기 위해 아키텍처 자체를 바꾸는 시도로서 **양자컴퓨팅이 등장했다**는 흐름이 흥미로웠다.  
- 0 또는 1이 아니라, 0과 1의 상태가 동시에 존재할 수 있다는 **중첩 개념**이 직관적으로는 이해하기 어렵지만, 이걸 잘 활용하면 우리가 익숙한 알고리즘 복잡도 개념 자체가 달라질 수 있다는 점이 인상적이었다.  
- 양자컴퓨터가 상용화되면 지금의 공개키 암호 체계가 위협받을 수 있다는 이야기를 들으면서, 기술 발전이 보안·정책·법 제도에 미치는 영향도 같이 따라온다는 점을 다시 한 번 느꼈다.  
- 강의에서 양자컴퓨터를 기존 컴퓨터의 대체제가 아닌 **특정 문제에 특화된 ‘가속기’ 같은 존재**로 보는 관점이, 현실적인 전망처럼 느껴져서 기억에 남았다.

## 4. 내 생각 / 느낀 점
- 양자컴퓨팅은 아직 현실에서 써볼 기회가 거의 없어서 SF 같은 기술로만 느껴졌는데, 실제로는 이미 여러 나라와 IT 기업들이 경쟁적으로 연구·투자를 하고 있다는 점이 놀라웠다.  
- 정보통신공학 전공 관점에서, 앞으로는 단순히 CPU/GPU만 이해하는 것이 아니라, **다양한 연산 자원을 네트워크로 묶어 서비스로 제공하는 구조**를 이해하는 것이 중요해질 것 같다고 느꼈다.  
- 양자컴퓨팅이 암호 해독, 최적화, 시뮬레이션 분야에 큰 영향을 줄 수 있다면, 데이터사이언티스트 입장에서는 어떤 문제를 양자컴퓨팅에 맡기고, 어떤 문제는 기존 방식으로 풀지를 설계하는 역할도 생길 것 같다는 생각이 들었다.  
- 아직은 추상적인 단계지만, 언젠가 양자컴퓨팅 관련 라이브러리나 클라우드 서비스를 써볼 수 있는 시점이 오면, 꼭 한 번 간단한 예제를 직접 돌려보고 싶다는 생각이 들었다.

## 5. 더 찾아보고 싶은 것 (데이터사이언티스트 관점)
- 양자컴퓨팅이 특히 강점을 가지는 것으로 알려진 **최적화 문제, 조합 탐색 문제**(예: 물류 경로 최적화, 포트폴리오 최적화 등)를 데이터사이언티스트 관점에서 어떻게 모델링하는지 공부해보고 싶다.  
- IBM, Google, AWS 등에서 제공하는 양자컴퓨팅 클라우드 서비스(시뮬레이터 포함)를 활용해, 간단한 양자 회로나 기본 양자 알고리즘(그로버, 쇼어 일부 개념)을 직접 실습해 보고 싶다.  
- 양자컴퓨팅이 본격적으로 상용화될 경우, **현재의 암호화·보안 체계와 데이터 보호 방식이 어떻게 바뀌어야 하는지**, 그리고 데이터사이언티스트가 이런 변화 속에서 어떤 역할(보안 분석, 리스크 평가 등)을 할 수 있을지 사례 중심으로 살펴보고 싶다.  
- 실제 기업/연구소에서 양자컴퓨팅과 머신러닝을 결합한 **양자 머신러닝(Quantum Machine Learning)** 연구가 어느 단계까지 와 있는지, 공개된 튜토리얼이나 논문을 통해 정리해보고 싶다.
