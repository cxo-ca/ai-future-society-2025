# [12주차] 예술과 인공지능

## 1. 오늘의 키워드
- 예술의 도구로서의 컴퓨터
- 창작하는 컴퓨터
- 미술 창작과 인공지능: 아카이빙, 인공지능 패턴, 창작 알고리즘
- 음악 창작과 인공지능: 스타일 학습, 예측 기반 작곡

## 2. 강의 핵심 내용 요약
- 먼저 예술이 어떻게 표현되는지 간단히 짚으면서, 물감·캔버스·사진기·카메라처럼 컴퓨터도 이제 예술가들이 사용하는 또 하나의 도구가 되었다는 점을 설명했다.
- 예술의 도구로서의 컴퓨터에서는, 그래픽 프로그램·3D 모델링·디지털 페인팅·영상 편집 도구 등 디지털 환경이 예술 작업의 전 과정에 스며들어 있다는 점을 다루고, 더 나아가 알고리즘 자체가 표현 수단이 될 수 있다는 관점을 소개했다.  
- 창작하는 컴퓨터 파트에서는, 인공지능이 기존 작품들을 학습해서 새로운 이미지를 생성하거나, 스타일을 변형하고, 전혀 새로운 조합을 만들어내는 생성 알고리즘을 다루며 인간과 기계가 협업하는 창작 방식의 가능성을 보여주었다.  
- 미술 창작과 인공지능 기술에서는  
  - 예술 작품의 아카이빙: 디지털 스캔과 메타데이터를 통해 방대한 작품들을 데이터베이스화하고, 이를 분석·검색·연구에 활용하는 방식  
  - 인공지능 패턴 적용하기: 스타일 변환, 패턴 분석, 색채·구도 분석 등 알고리즘을 활용해 기존 이미지에 새로운 시각 효과를 입히는 기술  
  - 창작 알고리즘: 규칙 기반, 확률 기반, 학습 기반 알고리즘을 활용해 새로운 이미지를 생성하는 기법  
  등을 소개했다.  
- 음악 창작과 인공지능 기술에서는  
  - 작곡가의 스타일을 학습한 인공지능이 특정 작곡가를 연상시키는 멜로디·화성을 생성하는 방식  
  - 기억·계산·예측을 통해 다음 음이나 코드 진행을 제안하면서, 인간 작곡가와 협업하는 도구로 활용되는 사례  
  를 통해 인공지능이 단순 자동 작곡기가 아니라, 창작을 보조하는 도구가 될 수 있다는 가능성을 설명했다.  
- 전체적으로 예술과 인공지능의 관계를, 인간을 대체하는 존재라기보다는 새로운 표현 방식과 실험을 가능하게 하는 플랫폼으로 보는 관점이 강조되었다.  

## 3. 인상 깊었던 포인트
- 예술의 도구로서 컴퓨터를 바라보는 시각이 인상 깊었다. 예전에는 컴퓨터가 예술을 '흉내 내는 것'처럼 느껴졌는데, 이제는 예술가가 코드를 붓처럼 사용해 새로운 형식의 작품을 만든다는 설명이 흥미로웠다.  
- 예술 작품의 아카이빙을 단순 보존이 아니라, 방대한 데이터셋으로 만들어 패턴을 분석하고 시대별 특징을 찾아내는 작업이라는 설명이 인상적이었다. '갤러리에서 감상하던 작품들이 데이터로도 다시 읽힌다'는 느낌이 새로웠다.  
- 작곡가의 스타일을 학습한 인공지능이 비슷한 분위기의 음악을 만들어내는 사례를 보면서, 내가 좋아하는 음악들의 패턴도 결국 데이터와 규칙으로 어느 정도 분석될 수 있겠다는 생각이 들었다.  
- 예술과 인공지능의 관계가 단순히 자동 생성에 그치지 않고 인간이 생각하지 못한 조합을 제안해주는 아이디어 도구가 될 수 있다는 점이 인상 깊었다.  

## 4. 내 생각 / 느낀 점
- 예술과 인공지능을 함께 보니, 데이터사이언티스트가 다루는 데이터가 꼭 숫자·표만이 아니라 이미지·음악·텍스트처럼 감성적인 데이터도 될 수 있다는 점이 흥미로웠다. 결국 이들도 패턴을 가진 데이터라는 점에서는 다르지 않다.  
- 한편으로는, 인공지능이 만들어낸 이미지·음악이 많아질수록 '진짜 창작자'가 누구인지, 데이터셋에 포함된 원작자들의 권리는 어떻게 보호할지 같은 문제도 중요해질 것 같다고 느꼈다.  
- 앞으로 데이터를 활용한 창작 도구가 더 많아질수록, 기술적인 구현 능력뿐 아니라, 어떤 데이터를 쓰고, 어떻게 가공해서 어떤 결과를 만들어낼지에 대한 감각이 중요해질 것 같았다.  
- 나중에 기회가 된다면, 내가 좋아하는 시각 이미지나 음악 데이터를 모아서 간단한 생성 모델을 만들어 보고 '데이터 기반 창작'이 어떤 느낌인지 직접 경험해보고 싶다는 생각이 들었다.  

## 5. 더 찾아보고 싶은 것 (데이터사이언티스트 관점)
- 공개된 예술 이미지 데이터셋이나 음악 데이터셋을 활용해 간단한 스타일 분석이나 클러스터링을 해보고, 장르·시대·작가별 패턴이 어떻게 나뉘는지 시각화해보고 싶다.  
- 이미지 생성, 스타일 변환, 음악 생성 등에서 사용되는 대표적인 모델들(예: CNN 기반 스타일 변환, RNN·Transformer 기반 음악 생성)이 어떤 구조로 동작하는지 튜토리얼 수준에서 정리해보고 싶다.  
- 예술 분야에서 인공지능을 활용할 때 저작권·데이터 수집·학습 데이터 편향 등의 문제가 어떻게 논의되고 있는지, 실제 사례와 함께 정리해보고 싶다.
- 장기적으로는 텍스트·이미지·음악 데이터를 함께 다루는 멀티모달 생성 모델이 예술 작업에 어떻게 쓰이는지 살펴보고, 데이터를 다루는 입장에서 어떤 새로운 직무나 협업 방식이 등장하는지 알아보고 싶다.
