# [02주차] 사물인터넷 기술과 우리 생활의 변화

## 1. 오늘의 키워드
- **사물인터넷(IoT, Internet of Things)**
- 스마트 홈 / 스마트 헬스케어 / 스마트 시티(팜) / 스마트 자동차
- 만물인터넷(IoE, Internet of Everything)
- 보안·프라이버시 이슈

## 2. 강의 핵심 내용 요약
- 사물인터넷(IoT)은 인터넷을 기반으로 모든 사물들을 연결하여 사람과 사물, 사물과 사물간의 정보를 상호소통하는 지능형 기술 및 서비스로 정의되었다.
- 과거에는 컴퓨터와 스마트폰만 인터넷에 연결되었다면, 이제는 가전제품, 자동차, 조명, 공장 설비, 도시 인프라까지 네트워크에 연결되면서 일상생활 전반이 '데이터를 계속 만들어내는 환경'으로 바뀌고 있다는 점을 설명했다.  
- 스마트 홈(예: 사물인터넷 에어컨, 사물인터넷 홈 CCTV), 스마트 헬스케어(웨어러블 기기), 스마트 시티(교통·환경 센서, 스마트 팜), 스마트 자동차(자율주행)와 같이 IoT가 적용된 사례를 통해, 편리함과 효율성이 어떻게 향상되는지 소개했다.  
- IoT의 핵심 구성 요소로 **센서 → 네트워크(5G, Wi-Fi 등) → 클라우드/서버 → 서비스/앱** 구조를 설명하며, 이 과정에서 생성되는 방대한 데이터가 빅데이터·AI 분석의 중요한 재료가 된다는 점을 강조했다.  
- 동시에, 기기들이 계속해서 데이터를 수집하고 전송하기 때문에 보안 취약점, 개인정보 유출, 감시 사회로의 위험 등 부정적인 측면도 함께 다루었다.

## 3. 인상 깊었던 포인트
- 예전에 단순히 “스마트 가전 = 편리한 기능이 많은 가전” 정도로만 생각했는데, 실제로는 센서·통신·데이터·앱이 모두 연결된 **하나의 시스템**이라는 설명이 인상 깊었다.  
- IoT 기기 한두 개가 아니라, 도시 전체 인프라에 IoT가 적용된 스마트 시티 사례를 보면서, 기술이 생활 수준이 아니라 **도시 운영 방식 자체**를 바꿀 수 있다는 점이 새롭게 느껴졌다.  
- 편리함 뒤에 항상 보안 문제가 따라온다는 이야기(해킹 당한 CCTV, 기본 비밀번호 그대로 쓰는 공유기 사례 등)를 들으면서, 연결된 기기가 많아질수록 보안·프라이버시 설계가 더 중요해진다는 걸 다시 느꼈다.

## 4. 내 생각 / 느낀 점
- 평소에 사용하는 스마트폰, 스마트워치, 무선 이어폰, 스마트 TV 등이 사실은 모두 IoT의 일부라는 설명을 듣고, 내가 이미 “IoT 환경 속에서 살고 있다”는 게 실감났다.  
- IoT 기기가 늘어나면 사용자 입장에서는 편리하지만, 동시에 내가 언제 어디서 무엇을 했는지가 계속 데이터로 남기 때문에, 기술 발전과 개인정보 보호 사이의 균형을 어떻게 맞출지 고민이 필요하다고 느꼈다.  
- 정보통신공학 전공 입장에서, 네트워크·프로토콜·무선통신 지식이 IoT 인프라를 설계하고 운영하는 데 직접적으로 쓰일 수 있다는 점이 흥미로웠다.  
- 나중에 데이터사이언티스트/엔지니어로 일하게 된다면, IoT에서 나오는 센서 데이터(시간·위치·상태 값 등)를 활용해 **이상 탐지, 예측 유지보수, 사용자 행동 분석** 같은 분석 모델을 만들어볼 수 있겠다는 생각이 들었다.

## 5. 더 찾아보고 싶은 것
- 실제 상용 스마트 홈/스마트 시티 사례와, 거기서 사용하는 센서·통신 기술 종류  
- IoT 보안 사고(해킹 사례)와 이를 막기 위한 인증·암호화 기술  
- 공장/제조 현장에서의 산업용 IoT(IIoT)와 예측 유지보수 사례  
- IoT 센서 데이터 분석에 자주 쓰이는 알고리즘과 데이터 처리 구조(스트리밍 처리 등)
