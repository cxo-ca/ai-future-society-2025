# [06주차] 알고리즘이 여는 세상

## 1. 오늘의 키워드
- 스케줄링(Scheduling)
- 튜링 머신(Turing Machine)
- 알고리즘(Algorithm)
- 프로그래밍 언어(Programming Language)
- 스크래치 프로그래밍(Scratch Programming)

## 2. 강의 핵심 내용 요약
- 스케줄링은 한정된 자원(CPU 시간, 메모리, 네트워크 등)을 여러 작업에게 어떻게 나누어 줄지 결정하는 과정으로, 운영체제(OS)의 프로세스 스케줄링, 공정의 작업 순서 결정, 서비스 대기열 관리 등 다양한 분야에서 핵심 역할을 한다는 점을 설명했다.  
- 튜링 머신은 영국 수학자 앨런 튜링이 제안한 **이론적인 계산 모델**로, 무한한 테이프와 읽기/쓰기 헤드, 그리고 상태 전이 규칙을 이용해 어떤 알고리즘이 계산 가능한지를 정의하는 개념이라는 점을 다루었다.  
- 알고리즘은 주어진 문제를 해결하기 위해 설계된 **명확한 절차/규칙의 집합**으로, 같은 문제라도 어떤 알고리즘을 선택하느냐에 따라 효율성과 결과가 크게 달라질 수 있음을 강조했다.  
- 프로그래밍 언어는 사람이 생각한 알고리즘을 컴퓨터가 이해하고 실행할 수 있도록 표현하는 형식 언어로, 고급 언어(파이썬, 자바 등)와 저급 언어(어셈블리) 개념을 간단히 소개하며 언어의 차이보다 **논리와 구조를 표현하는 힘**이 더 중요하다는 점을 언급했다.  
- 스크래치 프로그래밍 실습을 통해, 텍스트 코드를 직접 쓰지 않고도 블록을 조립하는 방식으로 반복, 조건, 변수 같은 기초 개념을 체험하며 **컴퓨팅 사고력(Computational Thinking)** 을 익히는 경험을 제공했다.  

## 3. 인상 깊었던 포인트
- 스케줄링을 단순히 '순서 정하기' 정도로 생각했는데, 실제로는 누가 먼저, 얼마나 오래, 어떤 기준으로 자원을 쓰게 할지 정하는 중요한 의사결정 문제라는 설명이 인상 깊었다.  
- 튜링 머신처럼 추상적인 개념이 오늘날 컴퓨터의 이론적 기초가 되었다는 사실이 흥미로웠고, 우리가 사용하는 모든 프로그램이 결국은 어떤 상태 전이와 규칙들의 모음이라는 생각이 새로웠다.  
- 알고리즘은 ‘코딩 잘하는 사람’만의 영역이 아니라, **문제를 단계별로 쪼개고 절차를 만드는 사고방식 자체**라는 설명이 와닿았다.  
- 스크래치 실습에서 블록 몇 개만 조합해도 간단한 애니메이션이나 인터랙션이 만들어지는 걸 보면서, 프로그래밍의 본질이 문법보다 **논리와 흐름 설계**에 가깝다는 걸 다시 느꼈다.

## 4. 내 생각 / 느낀 점
- 그동안 알고리즘이라고 하면 코딩 테스트 문제 풀이를 가장 먼저 떠올렸는데, 이번 강의를 통해 운영체제 스케줄링이나 서비스 설계 등 **현실 세계의 자원 배분 문제**도 모두 알고리즘의 영역이라는 걸 알게 되었다.  
- 데이터사이언티스트를 목표로 할 때도, 단순히 모델만 잘 만드는 것이 아니라 '어떤 순서로 데이터를 처리할지, 어떤 규칙으로 사용자에게 추천을 보여줄지' 같은 부분에서 알고리즘적인 사고가 필요하다는 것을 느꼈다.  
- 스크래치처럼 시각적인 도구로 논리를 표현해보는 경험이 텍스트 기반 언어(파이썬, SQL 등)를 배울 때도 큰 도움이 될 것 같다고 생각했다.  
- 앞으로 더 복잡한 기술을 배우더라도, 결국은 **문제 정의 → 절차 설계 → 결과 해석** 의 기본 흐름이 중요하다는 점을 기억해두고 싶다.

## 5. 더 찾아보고 싶은 것 (데이터사이언티스트 관점)
- 실제 데이터 파이프라인과 머신러닝 워크플로우(데이터 수집 → 전처리 → 학습 → 배포 → 모니터링)가 어떻게 스케줄링되는지, 예를 들어 Airflow나 Prefect 같은 도구를 사용한 사례를 살펴보고 싶다.  
- 추천 시스템, 광고 노출, 작업 큐 처리 등에서 사용되는 **스케줄링·우선순위 알고리즘**이 어떤 기준(공정성, 효율, 수익 극대화 등)으로 설계되는지 공부해보고 싶다.  
- 튜링 머신과 계산 복잡도 이론(시간·공간 복잡도)이 머신러닝 모델 선택과 파이프라인 설계에서 어떤 직관을 줄 수 있는지, 기초 이론을 정리해보고 싶다.  
- 스크래치처럼 블록 기반 환경이 아니라 파이썬이나 R 같은 언어로 **간단한 데이터 처리 알고리즘**을 구현해 보면서, 알고리즘 사고와 데이터 분석을 자연스럽게 연결하는 연습을 해보고 싶다.
