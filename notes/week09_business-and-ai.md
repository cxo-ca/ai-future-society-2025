# [09주차] 비즈니스와 인공지능

## 1. 오늘의 키워드
- 인공지능이 비즈니스에 미친 영향
- 비즈니스 프로세스 자동화
- 데이터 분석을 통한 인지적 통찰 제공
- 고객·직원 업무 지원
- 소매업에서의 인공지능 적용 (고객 요구 분석, 효율성 향상, 대화형 에이전트)
- 클라우드 기반 AI 서비스
- 비즈니스에 인공지능을 성공적으로 도입하기 위한 조건
- [자율실습] 온라인 쇼핑몰 상품 분류

## 2. 강의 핵심 내용 요약
- 인공지능이 비즈니스에 미친 영향으로, 반복적인 비즈니스 프로세스를 자동화하여 **시간·비용을 절감**하고, 사람의 단순 업무 부담을 줄여 더 고부가가치 작업에 집중할 수 있게 한다는 점을 설명했다.  
- 데이터 분석을 통한 인지적 통찰 측면에서는, 대량의 거래 데이터·로그 데이터를 분석해 고객 행동 패턴, 이탈 가능성, 수요 예측 등을 도출하고, 이를 바탕으로 의사결정을 지원하는 역할이 강조되었다.  
- 인공지능은 고객과 직원의 업무 지원 도구로도 활용되며, 챗봇·가상 상담원·업무용 어시스턴트 등을 통해 문의 응대, 문서 작성, 일정 정리 등 다양한 업무를 보조하는 사례가 소개되었다.  
- 소매업에서의 인공지능 적용 사례로는  
  - 고객 요구 분석: 구매 이력·검색 이력·장바구니 데이터를 활용한 개인화 추천, 세그먼트 분석, 이탈 위험 고객 탐지  
  - 효율성 향상: 재고 관리, 수요 예측, 가격 최적화, 물류·배송 경로 최적화  
  - 대화형 에이전트: 챗봇·음성 비서 등을 통한 24시간 고객 상담 및 간단한 주문 처리  
  등이 언급되었다.  
- 비즈니스에 인공지능을 적용하기 위한 노력으로,  
  - 단순히 기술을 도입하는 것을 넘어, 실제 비즈니스 문제에 맞는 **비즈니스 AI 적용 전략**을 수립하고,  
  - 클라우드 서비스(AI API, AutoML, ML 플랫폼 등)를 활용해 초기 인프라 부담을 줄이는 방법,  
  - 성공적인 인공지능 도입을 위해 데이터 인프라 구축, 조직 문화 변화, 인력 교육이 함께 필요하다는 점을 다루었다.  
- 자율실습에서는 Colab 온라인 코딩 환경에서 온라인 쇼핑몰 상품 데이터를 활용해,  
  - 기초 환경 설정(00 기초환경)  
  - 학습 데이터로 모델 학습(01 학습하기)  
  - 새로운 상품에 대한 카테고리 예측(02 예측하기)  
  를 수행하며, 실제 비즈니스 데이터와 닮은 문제를 직접 해결해 보는 경험을 제공했다.

## 3. 인상 깊었던 포인트
- 인공지능이 비즈니스에서 하는 역할이 단순히 '자동화'가 아니라, 데이터 분석을 통해 **사람이 놓치기 쉬운 패턴을 찾아주는 인지적 파트너**라는 설명이 인상 깊었다.  
- 소매업 사례에서, 고객 요구 분석·재고 최적화·대화형 에이전트가 따로 노는 기술이 아니라, 하나의 고객 경험을 만들기 위해 서로 연결된다는 점이 흥미로웠다.  
- 클라우드 AI 서비스 덕분에 이제는 거대한 인프라를 가진 기업이 아니어도, 작은 조직이나 스타트업도 인공지능을 비교적 쉽게 활용할 수 있다는 점이 인상적이었다.  
- '성공적인 인공지능 도입'이 결국 기술보다 조직·데이터·사람 문제라는 이야기, 그리고 잘못 적용된 인공지능은 오히려 업무를 더 복잡하게 만든다는 경고가 현실적으로 느껴졌다.

## 4. 내 생각 / 느낀 점
- 비즈니스와 인공지능을 같이 보니까, 데이터사이언티스트는 모델만 만드는 사람이 아니라 **비즈니스 문제를 데이터 관점에서 재정의하고, 풀 수 있는 형태로 바꾸는 역할**을 해야 한다는 생각이 들었다.  
- 소매업 사례는 온라인 쇼핑몰 상품 분류 실습과 바로 연결되어, 실제로 내가 만든 모델이 향후 추천·검색·카테고리 관리 등 다양한 기능의 기반이 될 수 있겠다는 상상을 해보게 했다.  
- 클라우드 기반 AI 서비스가 많아질수록, '모델을 직접 만드는 능력'뿐 아니라 '어떤 서비스를 왜 선택하고, 어떻게 조합해 설계할지'를 판단하는 능력이 중요해질 것 같다고 느꼈다.  
- 인공지능 도입이 실패하는 이유 중 상당수가 데이터 품질·조직 문화·목표 설정 문제라는 점에서, 나중에 일을 할 때도 기술 이야기만 하지 않고, 비즈니스 목표와 팀 상황을 함께 보는 시야를 갖고 싶다는 생각을 했다.

## 5. 더 찾아보고 싶은 것 (데이터사이언티스트 관점)
- 실제 e커머스 기업에서 사용하는 **상품 분류·추천·검색 랭킹 모델**의 구조와 데이터 흐름(로그 수집 → 피처 엔지니어링 → 모델 학습 → 온라인 서빙)을 사례 중심으로 공부해보고 싶다.  
- 클라우드 플랫폼(AWS, GCP, Azure 등)에서 제공하는 AI/ML 서비스들 중, 온라인 쇼핑몰 시나리오에 바로 적용할 수 있는 것들(추천, 예측, Vision API 등)을 정리해보고, 간단한 PoC를 해볼 수 있는 구조를 설계해보고 싶다.  
- 비즈니스 지표(KPI)와 모델 성능 지표(Accuracy, F1, AUC 등)를 어떻게 연결할지, 예를 들어 상품 분류 정확도가 몇 % 올라가면 클릭률·전환율·매출에 어떤 영향을 줄 수 있는지 가정해보는 연습을 해보고 싶다.  
- 이번 9주차 자율실습(온라인 쇼핑몰 상품 분류)을 확장해서, 실제 공개 상품 데이터셋을 사용해 **카테고리 분류 + 간단한 추천 모델**까지 이어지는 작은 포트폴리오 프로젝트를 만들어보고 싶다.
