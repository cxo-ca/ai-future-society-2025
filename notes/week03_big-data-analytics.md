# [03주차] 빅데이터 분석

## 1. 오늘의 키워드
- 빅데이터(Big Data)
- 빅데이터를 이용한 문제 해결 과정
- 트렌드 분석(Trend Analysis)
- 사회연결망 분석(Social Network Analysis)
- 데이터 예측 분석(Predictive Analytics)

## 2. 강의 핵심 내용 요약
- 빅데이터 분석은 단순히 데이터의 양이 많은 것을 의미하는 것이 아니라, **질문 정의 → 데이터 구축·수집·분석·시각화 → 데이터 의미분석 → 서비스 기획, 마케팅**하는 전체 과정이라고 설명했다.
- 데이터로 사회 현상을 이해하는 방법으로 트렌드 분석, 사회연결망 분석, 데이터 예측 분석의 세 가지 접근을 소개했다.
- **트렌드 분석**은 시간에 따른 검색량, 언급량, 판매량 등의 변화를 살펴보면서 특정 키워드나 이슈의 인기도·관심도를 파악하고, 이를 통해 사회적 분위기나 시장의 흐름을 읽어내는 방법이다.
- **사회연결망 분석(SNA)** 은 사람·집단·계정 등을 노드(node)로, 관계(친구, 팔로우, 대화 등)를 링크(link)로 보는 관점에서 네트워크 구조를 분석하고, 중심성이 높은 노드, 군집 구조 등을 통해 정보 확산 구조나 영향력을 이해하는 방법이다.
- **데이터 예측 분석**은 과거 데이터를 바탕으로 패턴을 찾아 미래의 값을 예측하는 것으로, 분류/회귀 같은 머신러닝 기법을 활용하여 '무엇이 일어날 것인가?'를 모델링하는 접근이다. (예: 매출 예측, 수요 예측 등)
- 강의 전체를 통해, 같은 사회 현상이라도 **트렌드 분석·연결망 분석·예측 분석 중 어떤 관점을 선택하느냐에 따라 보이는 패턴과 활용할 수 있는 인사이트가 달라진다**는 점을 강조했다.

## 3. 인상 깊었던 포인트
- 빅데이터라는 말을 막연하게 '데이터가 많다'로만 알고 있었는데, 실제로는 문제 정의부터 결과 해석까지 이어지는 **하나의 문제 해결 프로세스**라는 점이 인상 깊었다.
- 트렌드 분석, 사회연결망 분석, 예측 분석이 각각 다른 방법이지만, 결국은 **같은 현상을 다른 시각으로 바라보는 도구**라는 설명이 이해하기 쉬웠다.
- 특히 사회연결망 분석에서 중심 노드, 허브, 커뮤니티 구조를 보는 방식이 흥미로웠고, 단순히 '누가 팔로워가 많다'를 넘어 **네트워크 구조 전체를 보는 안목**이 중요하다는 점이 기억에 남았다.
- 예측 분석 파트에서는 과거 데이터를 이용해 미래를 어느 정도까지 예측할 수 있는지뿐 아니라, 예측이 틀렸을 때 그 한계와 책임을 어디까지로 볼 것인지까지 고민해야 한다는 점이 인상적이었다.

## 4. 내 생각 / 느낀 점
- 평소에도 뉴스나 검색 트렌드를 많이 보지만, 이번 강의를 통해 어떤 데이터를 기준으로 트렌드가 정의되는지를 생각하게 되어서, 앞으로는 단순 인기 순위가 아니라 **데이터가 어떻게 수집·가공되었는지**도 함께 보게 될 것 같다.
- 사회연결망 분석은 SNS나 메신저 같은 실제 사용 서비스와 연결되어 있어서, 내가 속한 네트워크 구조나 정보가 어떻게 퍼지는지에 대해 더 관심이 생겼다.
- 데이터 예측 분석은 타이타닉 과제처럼 과거 데이터로 미래 결과를 예측하는 문제와 바로 연결되기 때문에, 내가 앞으로 더 많이 연습해야 할 핵심 스킬이라는 생각이 들었다.
- 전반적으로, 빅데이터 분석은 기술 자체보다 **어떤 문제를 어떤 관점으로 풀지 선택하는 능력**이 중요하다는 느낌을 받았고, 데이터사이언티스트를 목표로 할수록 이 문제 정의 능력을 꾸준히 길러야겠다고 느꼈다.

## 5. 더 찾아보고 싶은 것 (데이터사이언티스트 관점)
- 실제 서비스/비즈니스에서 트렌드 분석, 사회연결망 분석, 예측 분석이 각각 어떤 문제에 사용되는지(예: 마케팅 캠페인, 추천 시스템, 이탈 예측 등) 사례를 찾아보고 싶다.
- 공개 데이터(검색 트렌드, SNS 관계 데이터, 공공 데이터 등)를 활용해서 **하나의 사회 현상**을 트렌드 분석·연결망 분석·예측 분석 세 가지 관점으로 각각 분석해보는 개인 프로젝트를 해보고 싶다.
- 예측 분석 모델을 만들 때, 단순 정확도뿐 아니라 **비즈니스 지표(KPI)와 어떻게 연결되는지**를 설계하는 방법(AUC, 리프트 차트, 이익 곡선 등)을 더 깊게 공부해보고 싶다.
- 데이터사이언티스트가 실제 업무에서 어떤 분석 기법을 선택할지 의사결정을 내릴 때, 기술적인 기준(성능, 데이터 조건)과 비즈니스적인 기준(해석 가능성, 실행 가능성)을 어떻게 균형 있게 고려하는지 현업 사례를 찾아보고 싶다.
