# 9주차 실습 – 온라인 쇼핑몰 상품 분류

> 군e러닝 「인공지능과 미래사회」 9주차  
> 비즈니스와 인공지능: 온라인 쇼핑몰 상품 분류

---

## 실습 내용 및 수행 방법 

### 실습 내용(목표)
- 온라인 쇼핑몰의 상품명·설명 데이터를 이용해, 각 상품이 속하는 카테고리를 자동으로 분류하는 인공지능 모델을 구현한다.  
- 이를 통해 상품 분류 자동화가 비즈니스 효율성과 사용자 경험에 어떤 영향을 줄 수 있는지 이해하는 것을 목표로 한다.

### 수행 방법 및 과정
- 강의에서 제공된 예시 상품 데이터(상품명, 카테고리)를 불러와 결측치, 카테고리 분포 등을 확인하며 간단한 EDA를 수행하였다.  
- 상품명 텍스트를 소문자 변환, 특수문자 제거, 토큰화 등의 전처리 과정을 거쳐 학습에 적합한 형태로 정제하였다.  
- 전처리된 텍스트를 Bag-of-Words 혹은 TF-IDF 방식으로 벡터화하여, 머신러닝 분류 알고리즘이 사용할 수 있는 수치형 피처를 생성하였다.  
- 지도학습(supervised learning) 기반 분류 모델(예: 나이브 베이즈 또는 로지스틱 회귀)을 사용해 학습 데이터를 학습시키고, 테스트 데이터에 대한 예측을 수행하였다.  
- 예측 결과를 실제 라벨과 비교하여 정확도(accuracy) 등의 성능 지표를 계산하고, 오분류 예시를 따로 살펴보며 오류 패턴을 분석하였다.

---

## 실습 결과

### 실습 결과 이미지
- 혼동 행렬(confusion matrix)과, 일부 상품에 대한 **실제 카테고리 vs 예측 카테고리** 비교 화면을 캡처하여 저장하였다.  
- GitHub에는 예시로 `images/confusion_matrix.png`, `images/sample_predictions.png`와 같이 결과 이미지를 함께 업로드하여 시각적으로 성능을 확인할 수 있도록 한다.

### 실습 결과 설명
- 전체 테스트 데이터 기준으로 모델은 비교적 양호한 정확도를 보였으며, 상품명이 뚜렷한 카테고리를 나타내는 경우(예: 전자기기 vs 식품 등)는 안정적으로 분류되었다.  
- 반대로 의류·패션 액세서리처럼 상품명이 유사하거나 애매한 카테고리에서는 서로 다른 카테고리끼리 자주 혼동되는 경향이 나타났다.  
- 혼동 행렬을 통해 어떤 카테고리 쌍에서 오분류가 많이 발생하는지 확인할 수 있었고, 이를 기반으로 데이터 자체의 모호함과 모델 한계를 분리해서 생각해 볼 수 있었다.  
- 규칙 기반의 수동 분류에 비해, 일관된 기준으로 대량의 상품을 자동 분류할 수 있다는 점에서 온라인 쇼핑몰 운영 효율성과 품질 관리 측면에서 유의미한 가능성을 확인하였다.

---

## 실습 결과 분석 

### 실습에 적용된 인공지능 원리
- 이번 실습은 정답 라벨(상품 카테고리)이 주어진 데이터를 사용해 모델을 학습시키는 **지도학습(supervised learning)** 기반의 텍스트 분류 문제이다.  
- 텍스트 데이터를 숫자 벡터로 변환하기 위해 Bag-of-Words / TF-IDF 기법을 사용하였으며, 벡터화된 입력을 바탕으로 각 카테고리의 소속 확률을 예측하는 분류 알고리즘을 적용하였다.  
- “텍스트 → 벡터 표현 → 분류 모델 → 카테고리 예측”으로 이어지는 전형적인 머신러닝 파이프라인을 경험해 봄으로써, 실제 서비스에서 사용하는 텍스트 분류 시스템의 기본 구조를 이해할 수 있었다.

### 본인 전공에서의 활용 방안
- 데이터사이언티스트 관점에서, 본 실습은 텍스트 데이터 전처리, 피처 엔지니어링, 모델 학습·평가까지 한 흐름으로 경험할 수 있는 좋은 예시이다.  
- 실제 서비스 환경에서는 상품 분류뿐 아니라, 검색 키워드 추천, 스팸 리뷰 필터링, 연관 상품/개인화 추천 등 다양한 기능에서 유사한 텍스트 분류·예측 모델을 활용할 수 있다.  
- 특히 상품 등록 단계에서 자동으로 적절한 카테고리와 태그를 제안하면, 운영 인력의 부담을 줄이고 데이터 품질을 높이는 데 기여할 수 있다.  
- 잘 정리된 카테고리·태그 데이터를 바탕으로 판매량, 클릭률, 전환율을 분석하면, 어떤 상품군에 마케팅 자원을 집중해야 하는지 판단하는 **데이터 기반 MD/비즈니스 전략** 수립에도 도움을 줄 수 있다.  
- 이번 경험을 통해 “모델 성능 수치”뿐 아니라, 모델이 비즈니스 지표와 사용자 경험에 어떤 영향을 주는지를 함께 고민하는 태도가 데이터사이언티스트에게 중요하다는 점을 체감하였다.
